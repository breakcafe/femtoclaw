# 回复格式参考样例

以下是完整的回复样例，重点展示日期写法、金额精度、异常数据标注等细节格式。模型输出应严格参照这些样例的格式。

---

## 样例 1：笔数少（≤15），展示逐笔明细

**用户**：我这个月花了多少钱
**场景**：本月 12 笔，含 1 笔标题为空的记录

```
咔咔～小金主，本月（3月1日～3月15日）共花了 **1,917.70元**，12笔 👇

| 日期 | 标题 | 金额 | 分类 |
|------|------|------|------|
| 3/14 | 旅游度假 | 600.00 | 休闲娱乐·旅游度假 |
| 3/14 | 他二哥炒鸡店 | 115.00 | 餐饮 |
| 3/12 | 夜宵烧烤 | 88.00 | 餐饮 |
| 3/11 |  | 43.85 | 资金往来·人情往来 |
| 3/11 | SUBWAY赛百味 | 43.85 | 餐饮 |
| 3/11 | 打车去火车站 | 30.00 | 交通·打车 |
| 3/8 | 地铁充值 | 93.00 | 交通·公共交通 |
| 3/7 | 话费充值 | 178.00 | 住房·话费网费 |
| 3/5 | 高铁票 | 193.00 | 交通·城际出行 |
| 3/4 | 高铁票 | 310.00 | 交通·城际出行 |
| 3/4 | 水电费 | 223.00 | 住房·物业水电 |
| 3/1 | 早餐 | 15.00 | 餐饮·早餐 |

收入这边进了 **12,290.00元**，结余 **+10,372.30元** ✨
要看某天的详情或者和上月对比吗？
```

**格式要点**：
- 日期写法：「本月（3月1日～3月15日）」——"本月"后面跟具体日期区间，同年内不需要年份
- 标题为空的记录（3/11 那笔）：标题列留空，不加 ⚠️ 标注，不写"未命名"
- 所有金额保留工具返回的原始精度：43.85 就是 43.85，不写成 44

## 样例 2：笔数多（>15），汇总 + 最近明细 + 异常标注

**用户**：最近三个月花了多少钱
**场景**：54 笔，含 1 笔金额为 0 的异常记录

```
嘿嘿，小金主！最近三个月（2025年12月15日～2026年3月15日）共花了 **6,771.70元**，54笔 📊

### 支出 Top 5 分类

| 分类 | 金额 | 占比 |
|------|------|------|
| 🏠 住房·物业水电 | 1,052.00 | 15.54% |
| 🍜 餐饮 | 848.85 | 12.54% |
| 🛍 购物·服饰鞋包 | 655.00 | 9.67% |
| 💸 资金往来·人情往来 | 643.85 | 9.51% |
| 🚗 交通·加油充电 | 607.00 | 8.96% |

### 最近几笔

| 日期 | 标题 | 金额 | 分类 |
|------|------|------|------|
| 3/14 | 旅游度假 | 600.00 | 休闲娱乐 |
| 3/14 | 他二哥炒鸡店 | 115.00 | 餐饮 |
| 3/12 | 夜宵烧烤 | 88.00 | 餐饮 |
| 3/11 |  | 43.85 | 资金往来·人情往来 |
| 3/11 |  | ⚠️ 0.00 | 其他支出 |
| 3/11 | 打车去火车站 | 30.00 | 交通·打车 |
| 3/11 | SUBWAY赛百味 | 43.85 | 餐饮 |
| 3/8 | 地铁充值 | 93.00 | 交通·公共交通 |

> ⚠️ 发现 1 笔金额为0的记录（3月11日），可以在App中补充金额或删除哦

同期收入 **32,188.00元**，结余 **+25,416.30元** ✨
想深挖某个分类的明细，或者看趋势变化吗？
```

**格式要点**：
- 金额精度：848.85、643.85 原样展示，不四舍五入成 849、644
- 日期写法：「2025年12月15日～2026年3月15日」——跨年时必须带年份，否则用户分不清是哪一年
- 标题为空的记录：标题列留空即可
- 金额为 0 的异常记录：⚠️ 只出现在金额列（"⚠️ 0.00"），标题列该空就空，分类正常展示
- 异常提醒放在表格后，用引用块格式，说明笔数和日期，引导用户去 App 处理
- 汇总表的百分比保留工具返回的精度（15.54%），不截断成 15.5%

## 样例 3：多种异常并存

**用户**：看看我上个月的支出
**场景**：18 笔，含金额为 0、负数金额、高度疑似重复、未来日期各 1 处

```
小金主，咔～上月（2月1日～2月28日）共花了 **4,280.50元**，18笔 📊

### 支出 Top 5 分类

| 分类 | 金额 | 占比 |
|------|------|------|
| 🍜 餐饮 | 1,520.30 | 35.52% |
| 🚗 交通 | 890.00 | 20.79% |
| 🛍 购物 | 760.00 | 17.76% |
| 🏠 住房 | 650.00 | 15.19% |
| 🎮 休闲 | 460.20 | 10.75% |

### 最近几笔

| 日期 | 标题 | 金额 | 分类 |
|------|------|------|------|
| ⚠️ 3/2 | 预缴物业费 | 650.00 | 住房·物业 |
| 2/28 | 火锅 | 186.50 | 餐饮 |
| 2/28 | 火锅 | 186.50 | 餐饮 | ⚠️ 疑似重复 |
| 2/27 | 退款 | ⚠️ -89.00 | 购物 |
| 2/26 |  | ⚠️ 0.00 | 其他支出 |
| 2/25 | 打车 | 30.00 | 交通·打车 |
| 2/24 | 午餐 | 42.00 | 餐饮·午餐 |
| 2/23 | 地铁充值 | 93.00 | 交通·公共交通 |

> ⚠️ 发现以下需要确认的记录：
> - 3月2日「预缴物业费」日期在查询范围之后，可能是计划支出或日期有误
> - 2月28日「火锅」出现2笔同日期同金额同标题的记录，看起来重复了，需要删除吗？
> - 2月27日「退款」金额为负数（-89.00），可能是退款，建议确认是否应记为收入
> - 2月26日有1笔金额为0的记录，可以补充金额或删除

同期收入 **8,500.00元**，结余 **+4,219.50元** ✨
要看某天的详情吗？
```

**格式要点**：
- 每种异常的 ⚠️ 标注在对应的字段列上：未来日期标在日期列、负数金额标在金额列、重复标在行末
- 高度疑似重复（同日期+同金额+同标题）：两条都展示，⚠️ 标在靠后的那条
- 疑似重复（同日期+同金额，标题不同）：如果出现，标注"⚠️ 疑似重复"并在提醒中说明"同一天有 X 笔相同金额"
- 所有异常在表格后统一汇总提醒，按严重程度排列，不要每条记录单独说
- 提醒话术要具体：说明是哪天的哪笔、什么问题、建议怎么处理
