# Skill B: 预算管家

预算相关的查询、计算、设置与调整。虽然流量只有约5%，但追问率高达89.2%——几乎每次预算相关的对话都需要追问，说明这个场景对用户来说很重要但AI的支持很差。

## 为什么追问率这么高

预算问题的本质是"数学计算+数据查询"，而LLM不擅长精确计算。用户问"每天还能花多少"，AI需要用（剩余预算÷剩余天数）来回答，但如果分母取错（月总天数vs实际剩余天数），结果就会偏差。另外，"重置预算是删除预算吗"出现了1,153次——这反映的是产品文案层面的认知混淆，AI需要用清晰的语言帮用户理解区别。

## 场景分流

| 用户想要的 | 处理方式 | 关键工具 |
|-----------|---------|---------|
| 查看当前预算执行情况 | 直接查询 | `compare_budget_vs_actual`（**默认查当月**，无需传日期参数） |
| 制定新预算（给了具体金额） | 调用预算草案工具 | `extract_budget_draft` |
| 制定新预算（模糊，如"按平时花销"） | 先查历史数据→再生成草案 | `analyze_expense_categories` → `extract_budget_draft` |
| 想要一个通用预算模板 | 对话引导，不调工具 | 无（对话形式） |
| 问预算功能怎么用 | 这属于功能客服 | `query_similar_faq` |

## extract_budget_draft的输出处理

调用`extract_budget_draft`后，将返回的预算草案数据提取出来，用Markdown表格清晰展示给用户，并询问用户是否确认。用户回复确认后，再执行预算创建。

展示格式示例：
```
📊 **预算草案**（2025年10月）

| 分类 | 预算金额 |
|------|---------|
| 生活费 | 3,000元 |
| 游戏 | 700元 |
| 周边 | 300元 |
| **合计** | **4,000元** |

确认这个预算方案吗？回复「确认」我就帮你设置～
```

## 容易混淆的概念

用户说"分类"时，可能指三个不同的东西：
- **分类体系**（标签管理）：用户想增删改查标签本身——这属于Skill C
- **预算体系**（月度总预算）：用户想设置或查看总支出上限——这属于本技能
- **分类预算**（为某个分类设上限）：同时涉及"钱"和"标签"——也属于本技能

简单判断：query里有"预算"→大概率找本技能；没有"预算"只问"怎么管理分类"→大概率找Skill C。

## 明确数字优先于历史数据

当用户同时给出具体数字和"参考历史"（如"参考上月花销，预算4500"），直接采纳4500。明确数字是用户的意志表达，历史数据只是参考背景。
