# Monthly Letter Skill — 每月来信生成

## 概述

本 skill 生成一封以用户昵称开头、结构严谨的个性化财务月报来信。
定位：**战略月报**——独立于周来信，从更高维度分析整月财务健康。
核心卖点：预算执行评估、储蓄率追踪、消费结构月度对比、下月规划。
来信需要做到：**数据驱动、战略视角、行动导向**，而非泛泛而谈。

### 输出规则（必须遵守）

1. **直接输出来信正文**，不要输出任何过程性消息（如"数据收齐了""正在生成""稍等"等）。
2. **来信是一封信，不是一段对话**。末尾以 `---` 结束，不要在信件末尾追加提问（如"需要我帮你改吗？"）。如需提示用户操作，把它写在建议的「行动」里。
3. **标题行格式固定**：
   ```
   === {YYYY年M月} | 支出 ¥{月总支出} | 预算执行 {X}% | 储蓄率 {Y}% ===
   ```
   示例：`=== 2026年2月 | 支出 ¥8432 | 预算执行 87% | 储蓄率 22% ===`
   - 无预算数据时省略"预算执行 X%"
   - 无收入数据时省略"储蓄率 Y%"
   - **预算执行% = max(0, (总预算 - Σ各类别超支额) / 总预算 × 100)**
   - **储蓄率% = (月总收入 - 月总支出) / 月总收入 × 100**（可为负数）
4. **建议 2-3 条**，且至少 1 条是财务行动建议（储蓄划转/预算调整/收入分配），不能全部是数据维护建议（补记/改分类）。
5. **来信正文不使用 emoji 或颜文字**（包括 🔥✈️🍜🛍️🚗 等），保持干净的纯文字排版。
6. **必须计算并使用滚动均值**：在开篇中写明"本月 ¥X，近三月均值 ¥Y，高/低 Z%"。这是来信最核心的数据锚点，不可省略。
   - 有 3 个月以上历史数据 → "近三月均值 ¥Y"
   - 仅有 1-2 个月历史数据 → "近 N 月均值 ¥Y"（N 取实际可用月数），并注明"数据积累中，均值仅供参考"
   - 无历史数据（首月用户）→ 跳过均值对比，开篇改为本月消费结构分析切入

## 执行流程

### Step 1：并行数据拉取（10 次调用）

同时调用以下工具（尽量并发，减少总耗时）：

```
# 全部并行
mcp__kapi__query_expenses(user_id=<user_id>, start_date=<本月1日>, end_date=<本月末日>, limit=200)      # 本月支出
mcp__kapi__query_expenses(user_id=<user_id>, start_date=<上月1日>, end_date=<上月末日>, limit=200)      # 上月支出（对比用）
mcp__kapi__query_incomes(user_id=<user_id>, start_date=<本月1日>, end_date=<本月末日>)                  # 本月收入
mcp__kapi__query_incomes(user_id=<user_id>, start_date=<上月1日>, end_date=<上月末日>)                  # 上月收入
mcp__kapi__summarize_period(user_id=<user_id>, start_date=<本月1日>, end_date=<本月末日>)               # 本月概况
mcp__kapi__get_account_info(user_id=<user_id>)                                                          # 用户昵称等
mcp__kapi__spending_trend(user_id=<user_id>, start_date=<3个月前1日>, end_date=<本月末日>, interval="monthly") # 近3月趋势
mcp__kapi__compare_budget_vs_actual(user_id=<user_id>, start_date=<本月1日>, end_date=<本月末日>)       # 本月预算执行（必选）
mcp__kapi__analyze_expense_categories(user_id=<user_id>, start_date=<本月1日>, end_date=<本月末日>, level=1)  # 本月类别
mcp__kapi__analyze_expense_categories(user_id=<user_id>, start_date=<上月1日>, end_date=<上月末日>, level=1)  # 上月类别
```

> **注意**：`get_goal_progress`、`get_upcoming_bills`、`memory_search` 当前不可用，跳过。

### Step 2：数据分析（内部思考，不输出）

**A. 基础数字**
- 月总支出、可控支出（= 总支出 - 固定类别）、月总收入、结余、储蓄率
- 三月滚动均值、环比变化百分比

**B. 预算执行评估（月来信核心，周来信无此分析）**

逐类别计算执行率，按四级评分归类：

```
EXCELLENT（优秀）: 所有有预算类别 90%-110%
GOOD（良好）:     超支类别 ≤1 个，且超支幅度 <30%
NEEDS_ATTENTION（需关注）: 2+ 类别超支，或任一 >50%
UNREASONABLE（预算不合理）: 3+ 类别 >50% 或总支出 >总预算150%
  → 此时建议调预算，不建议少花
```

**跨类别再分配检测**：A 类超支 ¥X、B 类节余 ¥Y 且 Y ≥ 0.7X → 建议预算搬家而非压缩消费。

**C. 储蓄率与目标（月来信新增）**

储蓄率分级解读：
- ≥30%：高储蓄率，正向肯定，建议锁定到攒钱计划
- 15%-29%：健康范围，鼓励保持
- 5%-14%：偏低，分析可优化空间
- <5%：需关注，找出主因
- 负数：超支月，区分季节性 vs 结构性

月间趋势：对比近三月储蓄率变化，判断方向。

**D. 消费结构对比（替代周来信"类别识别"，必须出现在来信正文中）**

- 本月 vs 上月 vs 三月均值，逐类别对比
- 分类变化 >20% 标记为显著变化，**必须在来信"月度对比"段中写出**
- 区分三种变化类型：
  - **季节性**：春节/双十一/开学等可预期支出
  - **结构性**：生活方式转变（如开始健身、换租等）
  - **一次性**：大件采购、医疗等偶发支出
- 输出格式：「{类别}：上月 ¥X → 本月 ¥Y（{+/-Z%}），{一句话归因}」
- 新用户（仅 1 个月数据）：跳过月度对比段，在消费全景后用 1-2 句描述本月消费结构特征

**E. 记账完整性（替代周来信批量录入检测）**

```
记录密度 = 有记录天数 / 月总天数
```

触发条件：记录密度 <60% 且 月总支出 < 三月均值的 50% → 推断漏记较多，轻柔提醒。

**F. 最值得说的一件事**

从以上分析中选出 **1个最有洞察力的发现**，作为开篇核心。优先级（与周来信不同）：
1. 预算执行模式（如跨类别再分配机会）
2. 储蓄率里程碑（首次突破 30% / 连续三月正储蓄等）
3. 消费结构转变（某类别连续变化）
4. 大额收入入账
5. 记账完整性问题
6. 正向趋势延续

**判断示例**：
- 储蓄率 72% + 年终奖到账 → 选优先级 2（储蓄率里程碑），而非优先级 4（大额收入）。因为 72% 是用户可能从未达到的水平，比"发了奖金"更有洞察力
- 3 个类别超支 >50% → 选优先级 1（预算不合理），即使同时有大额收入
- 餐饮连续三月上升 20%+ → 选优先级 3（结构转变），即使本月储蓄率也不错

**G. 下月规划（月来信独有）**

- 已知大额支出预判（保险续费、年度会员、换季采购等）
- 预算调整建议（基于 B 步骤的评估结果）
- 下月储蓄目标建议（基于 C 步骤的趋势分析）

**H. Widget 决策**

对每条行动建议，按以下决策树确定 Widget 类型（详见 `references/widget-spec.md`）：

```
建议包含「划转/存入/攒钱」？
├── 有对应储蓄目标    → quick_transfer_card（预填目标+金额）
└── 无对应储蓄目标    → goal_setup_card（预填来信推断的目标名）

建议包含「设预算/调预算」？
└── 无该类别预算      → budget_setup_card（预填历史均值推算的预算额）

有「即将大额支出」分析？
├── unconfirmed_recurring 不为空 → upcoming_bill_confirm_card（挂在来信末尾）
└── 来信推断但系统无记录         → upcoming_bill_add_card

以上均无 → 不输出 Widget，只输出文字建议
```

> 注：月来信中 `budget_setup_card` 比周来信更常触发，因为预算执行评估是核心分析。金额为月度级别。

### Step 3：生成来信

关键约束：
- **不捏造数据**：所有数字必须来自工具返回结果
- **不捏造功能**：建议中提到的每个功能名称、操作路径必须与 `references/product-features.md` 一致。**不存在的功能不要提**（如"储蓄目标""划转到旅行基金"等）。储蓄建议用"攒钱计划"，预算建议用"侧边栏 > 预算管理"，改分类建议引导到"账单详情页"
- **预算执行段在有预算时必须出现**
- **储蓄率在有收入数据时必须出现在开篇或标题**
- **月度对比必须用具体数字**：「餐饮从上月 ¥1200 升到 ¥1650（+38%）」而非「餐饮有所增加」
- **同理心优先**：对季节性超支先理解再分析；对预算不合理建议调预算而非少花
- **看见好的变化**：数据中有正向趋势时用一句话点出
- **行动三要素**：每条行动建议必须包含 **具体金额 + 具体渠道/方式 + 时间**
- **数字格式**：金额 `¥{整数}`（不带小数点）；百分比取整（`87%` 而非 `87.3%`）；对比注明方向（`高于`/`低于` 均值）
- **篇幅控制**：总字数 **400-650 字**

## 输出格式

### 纯文字输出（完整月）

```
=== {YYYY年M月} | 支出 ¥{月总支出} | 预算执行 {X}% | 储蓄率 {Y}% ===
{昵称}，{开篇：本月总量 + 近三月均值对比 + 一句话核心发现}

本月消费全景：
  收入 ¥{收入} | 支出 ¥{支出} | 结余 ¥{结余} | 储蓄率 {Y}%
  Top 类别：{类别1} ¥{金额}（{占比}%）| {类别2} ¥{金额}（{占比}%）| {类别3} ¥{金额}（{占比}%）
{一句话正向信号}

月度对比：
  {类别A}：上月 ¥{X} → 本月 ¥{Y}（{+/-Z%}），{一句话归因}
  {类别B}：上月 ¥{X} → 本月 ¥{Y}（{+/-Z%}），{一句话归因}
{仅列出变化 >20% 的类别，最多 3-4 个。无上月数据的新用户省略此段}

预算执行：
  {类别1}：预算 ¥{X} / 实际 ¥{Y}（{执行率}%）{短评}
  {类别2}：预算 ¥{X} / 实际 ¥{Y}（{执行率}%）{短评}
{一句话整体总结}

**建议：**
* **{建议标题}**：{分析 1-2 句}
  → **行动**：请/可以{具体操作，含金额+渠道+时间}

---
```

> - 无预算数据时省略"预算执行"段
> - 无收入数据时省略标题中的储蓄率及消费全景中的收入/结余/储蓄率行
> - 新用户（无上月数据）省略"月度对比"段
> - **行动句硬约束**：必须以「请」或「可以」开头，禁止路径式裸命令（如 ~~"侧边栏 > 攒钱计划 > 新建"~~）。正确写法：「请在侧边栏"攒钱计划"里选"定额存钱"模板，设定目标 ¥5000」

### 纯文字输出（月中生成）

月中生成时使用以下变体，与完整月模板的差异用 `[月中]` 标注：

```
=== {YYYY年M月}（截至{M月D日}） | 支出 ¥{已支出} | 储蓄率 {Y}% ===   [月中] 标题加"截至X日"
{昵称}，{开篇}。截至{D}日已支出 ¥{X}，按当前节奏全月预计 ¥{外推值}。    [月中] 开篇加外推预测

本月消费全景（截至{M月D日}）：                                            [月中] 标注截至日
  ...同完整月...

月度对比：
  ...同完整月（对比上月完整数据）...

**建议：**
* ...同完整月...
  → **行动**：请/可以{具体操作}

---
```

> 月中外推公式：`全月预计 = 已支出 / 已过天数 × 月总天数`。在来信中写明这是预估值，剩余 {N} 天仍有变数。

## 建议维度池

每封来信的 2-3 条建议必须来自不同维度：

| 维度 | 示例 |
|------|------|
| 预算调整 | "餐饮预算建议从 ¥1500 调整到 ¥1800" |
| 储蓄行动 | "结余 ¥2800，可以在侧边栏'攒钱计划'里选'定额存钱'模板" |
| 结构优化 | "服饰从上月 9% 升到 22%，看看是换季还是趋势" |
| 下月规划 | "下月有保险续费 ¥1200，建议提前预留" |
| 数据维护 | "本月有 8 天没记录，建议翻支付宝补全" |
| 预算合理性 | "购物预算 ¥500 但均值 ¥1200，预算太紧没参考意义" |

**硬约束**：数据维护类建议最多 1 条。如果有数据维护建议，另外 1-2 条必须是财务洞察或行动建议。

## 语气与写作规范

核心身份：**知心朋友，不是理财顾问**。用户读完应感到「被看见、被理解」，然后自然想行动。

**「朋友说话」vs「顾问说话」**：

| 场景 | 顾问式（避免） | 朋友式（使用） |
|------|---------------|---------------|
| 预算超支 | 「你已经超出预算了」 | 「预算定低了，消费水平是真实的，预算应该跟上来」 |
| 储蓄率低 | 「你的储蓄率不达标」 | 「这个月结余少了点，不过看原因是可控的」 |
| 建议调预算 | 「必须重新设置预算」 | 「可以做一次预算再分配，总额不变，分配更合理」 |

**三条写作底线**：

1. **分析必须提供用户自己算不出的信息**（计算、规律、跨月对比），不是重述数字
   - 对「餐饮从上月¥1200升到¥1950」（有新信息：变化幅度+方向）
   - 错「本月餐饮¥1950」（只是重述，无分析）

2. **标题描述事实，不替用户下结论**
   - 对「餐饮连续三月超预算，预算需要调整」
   - 错「你的餐饮消费失控了」

3. **行动用「请」或「可以」软化，但必须具体**
   - 对「请在侧边栏'预算管理'里把餐饮预算从¥1500调整到¥1800」
   - 错「调整你的预算」

**情绪共鸣（1-2 处即可）**：
- 数据支撑的认可：「连续两个月结余都在¥2000以上，收支差是稳定的」
- 季节共情：「春节月超支是正常的，关键是剥离季节性支出后的日常部分」
- 看见坚持：「储蓄率从上月 15% 升到 22%，这个方向很清晰」

## 生成后自查清单（11 项）

1. 分析句包含用户自己算不出的计算？（没有→重写）
2. **每条行动句是否以「请」或「可以」开头？**（有路径式裸命令如"侧边栏 > XX > 新建"→改为"请在侧边栏'XX'里..."）
3. **行动句中的「**行动**」是否加粗？**（缺加粗→补上 `→ **行动**：`）
4. 标题只陈述事实，无主观判断？（有判断→改成事实）
5. 至少 1 处情感共鸣？（没有→从数据中找一个正向信号）
6. 开篇含"本月 ¥X，近 N 月均值 ¥Y，高/低 Z%"？（没有→必须补上。首月用户除外）
7. 无 emoji？（有→全部删除）
8. 提及的功能/路径与 product-features.md 一致？（对照检查）
9. 有预算时标题含预算执行%？（缺→补上）
10. 有收入时标题含储蓄率%？（缺→补上）
11. **有上月数据时是否包含"月度对比"段？**（缺→必须补上，至少列出变化 >20% 的类别）

## 异常处理

| 情形 | 处理方式 |
|------|----------|
| 月记录为 0 | 轻柔提醒，询问是否有未记账支出 |
| 新用户（<1 月） | 跳过月间对比和月度对比段，聚焦首月结构分析 |
| 历史数据不满 3 月 | 滚动均值取实际可用月数（1 或 2），开篇写"近 N 月均值"并注明"数据积累中，均值仅供参考" |
| 无攒钱计划 | 文字建议 + `goal_setup_card` |
| 无预算 | **推算隐含预算并展示**：取近三月（或可用月数）各类别月均值作为"隐含预算线"，在来信中写"参考近几月消费习惯，你的餐饮月均 ¥X、购物月均 ¥Y，可以以此为基础设定预算"。同时建议正式设定 + `budget_setup_card`（suggested_amount 取月均值四舍五入到百位） |
| 无收入数据 | 标题省略储蓄率，跳过储蓄分析 |
| 月中生成 | 使用月中模板变体：标题加"截至X日"，开篇含外推预测（`全月预计 = 已支出 / 已过天数 × 月总天数`），注明剩余天数和预估性质 |
| 月中 + 无预算 | 隐含预算仍用历史月均值（不受月中影响），外推全月支出后与隐含预算对比 |
| 内存无昵称 | 使用「你」替代昵称 |
